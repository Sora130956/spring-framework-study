# Spring Framework Study Notes — 设计规范

## 目标

本仓库用于 Spring 官方文档的精读学习，兼顾：
1. 通过文档原文学习技术英语
2. 深度学习 Spring Framework 技术细节

终极目标：为转型海外 freelancer 积累技术实力和英语能力。

## 仓库结构

```
spring-framework-study/
├── README.md
├── templates/
│   └── note-template.md
├── core/
│   ├── 01-ioc-container/
│   │   ├── container-overview.md
│   │   └── bean-overview.md
│   └── ...
├── aop/
├── mvc/
└── boot/
```

- 按 Spring 模块分顶层目录（core、aop、mvc、boot...）
- 模块下按文档章节分子目录，子目录使用 `序号-英文短名` 命名
- 每篇笔记一个 md 文件，文件名使用章节英文原名（不做翻译）
- 学习顺序：Core → MVC → Boot → 其余模块

## 单篇笔记结构

```markdown
# <章节英文标题>

> **来源:** <Spring 官方文档 URL>

## 核心理解

<!-- 中文总结本节核心概念，2-3 段 -->

## 关键点

### <知识点一>

> 原文引用

中文理解 + 代码示例

### <知识点二>

...

## 句子解析

### 原文: "..."

- **翻译:** ...
- **解析:** 主句结构 / 关键短语 / 值得注意的用法

## 术语表

| 英文 | 词性 | 释义 |
|------|------|------|
| instantiate | v. | 实例化 |
```

字段说明：
- **核心理解**：用自己的话概括本节讲了什么，2-3 段中文
- **关键点**：原文引用 + 中文理解 + 代码示例，每个知识点一个小节
- **句子解析**：挑选 2-3 个理解困难的句子，做翻译和语法/句式分析
- **术语表**：只记录生僻、专业的词语；跳过 CET-4 (612分) 范围内已掌握的通用词汇

## 工作流

1. 用户在阅读 Spring 官方文档时，复制关键段落并附简短中文备注
2. 用户将文档 URL + 复制的内容 + 理解发给 Claude
3. Claude 将这些原始笔记整理为结构化笔记，放入对应模块目录
4. Claude 按术语表筛选标准，从文中提取生僻/专业术语填入术语表

## 术语筛选标准

- **收录**：技术专业术语（如 autowire, aspect, pointcut）、低频学术词汇
- **跳过**：CET-4 612 分范围内的通用词汇（常见动词、名词、形容词等）
- 判断标准：该词是否在通用英语中常见？是否属于 Spring/Java 特有概念？
